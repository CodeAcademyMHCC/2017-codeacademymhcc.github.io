Cheatsheets

This page contains quick reference "cheatsheets" we can use as we go along.

  • Console
  • Git
ls
List the contents of a directory. Example: ls. If you want to show all files ending with a specific extension you can specify that like so: ls *.html or vice-versa ls index.*
cd
Change the working directory. Example: cd css
mv
Move a file from one location to another.Example: mv style.css css/style.css
mkdir
Create a new directory. Example: mkdir css
rm
Remove a file. Example: rm style.css
pwd
Show the current working directory. Example: pwd → /home/user/public_html/
clear
Clear the console screen

git init
Initialize a new git repository in your current working directory.
git clone user@server:repository-path .
Clone a remote URL into the current working directory, If the "." is omitted it clones the repository into a new directory relative the one you are in.
git add .
Stage all untracked files for your next commit. You can optionally name specific files and folders to add.
git commit -m "commit message"
Commit staged files into the repository history.
git branch
Show the current branch you're on.
git checkout (branch)
Checkout the specific branch. Adding a -b flag will create the branch if it does not already exist.
git push origin (branch)
Push your committed change to the origin remote branch
git fetch
Brings your local branches up to date with the remote branches.
git pull
Brings brings remote changes into your currently checked out branch.
